About this project

While there exists extensive studies and surveys for state or nationwide medical issues, there are few tools available for local practices to use to analyse their own personal set of data. As each practice supports a region of different climates and cultural backgrounds, the typical type of cases they encounter may differ from practice to practice. From my research, while local practices have access to a wealth of data, they do not have the tools available to them to process this data.

As a result, this tool will process this medical data and offer the user an easy to use interface which can break down data and identify trends. This app will be capable of accepting medical data from local medical practices, visualising it and identifying trends. Initially, the tool would accept data from an excel sheet with anonymised records of patients. A basic data sheet would include the date of appointment, their symptoms and any diagnosis. Columns will be used to separate multiple symptoms and each row would represent the instance of the patients visit.

This tool is aimed to be a low cost and lightweight solution which would give small practises big data processing capabilities normally only afforded to large scale hospitals and dedicated analyst firms
